# Contributing to Desktop Automation Recorder ⚡

First off, thank you for considering contributing to AutoRecorder! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## Code of Conduct

By participating in this project, you agree to abide by our Code of Conduct.

## How Can I Contribute?

### Reporting Bugs
- Use the [GitHub issue tracker](https://github.com/waveWhirlVfx/DesktopAutomationRecorder/issues).
- Check if the bug has already been reported.
- Use a clear and descriptive title.
- Provide as much detail as possible (macOS version, steps to reproduce, expected vs actual behavior).

### Suggesting Enhancements
- Open an issue with the "enhancement" tag.
- Explain the "why" behind the suggestion.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the build passes.
4. Issue a pull request!

## Style Guide
- Follow [Swift API Design Guidelines](https://swift.org/documentation/api-design-guidelines/).
- Keep functions small and focused.
- Comment complex logic.
